# Проверка заклёпок на брак

Веб-приложение для автоматического поиска бракованных заклёпок на металлических
деталях по фотографиям и видео. Пользователь загружает файлы, нажимает «Анализ» —
система находит каждую заклёпку, классифицирует её (годная / брак) и считает
процент брака по детали, с визуализацией и историей анализов.

Подход — **transfer learning** на основе детектора **YOLO26** (Ultralytics).

## Возможности

- Загрузка до **4 фото** и **4 видео** за один анализ.
- Детекция каждой заклёпки рамкой + классификация (годная / брак).
- Честный **процент брака** = доля бракованных среди целых заклёпок.
- Исключение **обрезанных краем кадра** заклёпок из расчёта.
- Визуализация: рамки и номера на фото, круговая диаграмма, график уверенности
  по каждой заклёпке, сводная диаграмма по файлам (matplotlib).
- Обработка **видео**: сэмплирование по времени (кадр раз в 0.5 с, первые 10 с).
- **История анализов** в PostgreSQL.

## Установка и запуск

**Требуется:** Python 3.10, PostgreSQL.

```powershell
# 1. Окружение и зависимости
py -3.10 -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt

# 2. Секреты БД: скопировать шаблон и вписать пароль PostgreSQL
Copy-Item .env.example .env
# открыть .env и заполнить DB_PASSWORD

# 3. Создать базу данных (один раз). Например через psql:
#    psql -U postgres -c "CREATE DATABASE rivet_analyzer;"

# 4. Положить обученные веса в models/best.pt

# 5. Запуск сервера
uvicorn app.main:app --reload
```

Открыть в браузере: **http://127.0.0.1:8000/**
Интерактивная документация API: **http://127.0.0.1:8000/docs**

**Таблицы** создаются автоматически при первом запуске. Саму базу `rivet_analyzer`
нужно создать один раз заранее (шаг 3) — SQLAlchemy создаёт только таблицы, не базу.

## Как считается процент брака

`процент брака = бракованные / все целые заклёпки × 100`

Считаются только **целые** заклёпки (обрезанные краем кадра исключаются, так как
их нельзя оценить достоверно). Показатель «средняя уверенность модели» — это
уверенность детектора, **а не** гарантия правильности.

## Данные и обучение

- Датасеты заклёпок собраны с Roboflow Universe, классы приведены к схеме
  `rivet_ok` / `rivet_defect`, применена аугментация.
- Обучение — **YOLO26** в Google Colab (бесплатный GPU); ноутбук используется
  только для инференса готовых весов.

## Ограничения и развитие

- Обучающий датасет небольшой и не авиационного домена → на нестандартных кадрах
  (реальная обшивка, царапины, надписи) возможны **пропуски** и **ложные
  срабатывания** (domain shift). Метрики валидации оптимистичны (малая выборка).
- Видео считается покадрово без трекинга — одна заклёпка учитывается в нескольких
  кадрах (процент остаётся осмысленным, абсолютные числа завышены).

**Направления развития:** больше разнообразных данных (в т.ч. реальная обшивка) и
переобучение; трекинг объектов в видео; калибровка порогов по разнородной валидации.
