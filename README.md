# Проверка заклёпок на брак

Веб-приложение для автоматического контроля заклёпок на металлических деталях по
фотографиям и видео. Два режима:

1. **Деталь — несколько заклёпок** (детекция): на фото/видео много заклёпок —
   система находит каждую рамкой, классифицирует (годная / брак) и считает
   **процент брака** по детали.
2. **Одна заклёпка — несколько ракурсов** (классификация): несколько фото или
   видео одной заклёпки с разных камер/ракурсов — система выдаёт **один вердикт**
   (вероятность брака по худшему ракурсу).

Подход — **transfer learning** на моделях **YOLO26** (Ultralytics): детектор
(`best.pt`) и классификатор (`best_cls.pt`).

## Возможности

- Два режима: детекция многих заклёпок и классификация одной заклёпки по N ракурсам.
- Загрузка фото и видео (лимит настраивается в `config.py`).
- Честный **процент брака** = доля бракованных среди целых заклёпок.
- Исключение **обрезанных краем кадра** заклёпок из расчёта.
- Вердикт по заклёпке из нескольких ракурсов — **по худшему** (достаточно одной
  камеры, увидевшей дефект).
- Визуализация: рамки и номера на фото, круговая диаграмма, график уверенности,
  сводная диаграмма по файлам (matplotlib).
- Обработка **видео**: сэмплирование по времени (кадр раз в 0.5 с, первые 10 с).
- **Полоска прогресса** «n из m».
- **История анализов** в PostgreSQL (отдельно для детекции и для заклёпок).
- **Логирование** в файл `logs/app.log` (с ротацией) и в консоль.
- **Автовыбор устройства**: CUDA (видеокарта), если доступна, иначе CPU.
- **Docker** — развёртывание одной командой (CPU и GPU варианты), с PostgreSQL.

## Запуск локально

**Требуется:** Python 3.10, PostgreSQL.

```powershell
py -3.10 -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt

Copy-Item .env.example .env          # вписать DB_PASSWORD
psql -U postgres -c "CREATE DATABASE rivet_analyzer;"   # создать базу один раз
# положить веса в models/best.pt и models/best_cls.pt

uvicorn app.main:app --reload
```
Открыть **http://127.0.0.1:8000/**, документация API — **/docs**.
Таблицы создаются автоматически при первом запуске.

## Запуск в Docker (одной командой)

**CPU:**
```powershell
docker compose up --build
```
**GPU** (нужен NVIDIA-драйвер + поддержка GPU в Docker):
```powershell
docker compose -f docker-compose.yml -f docker-compose.gpu.yml up --build
```
Открыть **http://localhost:8000/**. PostgreSQL поднимается своим контейнером
(пароль берётся из `.env`).

## Вычисления на GPU

Приложение само определяет устройство (`config.DEVICE`): если стоит CUDA-сборка
PyTorch и видна видеокарта — считает на GPU (`cuda:0`), иначе на CPU. Поддерживается
несколько видеокарт (`DEVICE=0,1`). Для GPU локально поставить CUDA-сборку torch:
```powershell
pip install torch torchvision --index-url https://download.pytorch.org/whl/cu121
```

## Как считается результат

- **Детекция:** `процент брака = бракованные / целые заклёпки × 100` (краевые
  исключаются). «Средняя уверенность модели» — уверенность детектора, не гарантия.
- **Классификация:** для каждого ракурса модель даёт вероятность брака; итог по
  заклёпке — **максимум** (по худшему ракурсу), вердикт БРАК если ≥ порога
  (`DEFECT_VERDICT_THRESHOLD`).

## Данные и обучение

- Датасеты заклёпок собраны с Roboflow Universe, классы приведены к схеме
  `rivet_ok` / `rivet_defect`, применена аугментация.
- **Детектор** обучен на размеченных рамками фото; **классификатор** — на
  вырезанных по рамкам одиночных заклёпках (датасет получен кропом из детекционного).
- Обучение — в Google Colab (бесплатный GPU); ноутбук используется для инференса.

## Ограничения и развитие

- Обучающий датасет небольшой и не авиационного домена → на нестандартных кадрах
  (реальная обшивка, царапины, надписи) возможны **пропуски** и **ложные
  срабатывания** (domain shift). Метрики валидации оптимистичны (малая выборка).
- Видео считается покадрово без трекинга — одна заклёпка учитывается в нескольких
  кадрах (процент осмыслен, абсолютные числа завышены).

**Направления развития:** больше разнообразных данных (реальная обшивка) и
переобучение; трекинг объектов в видео; калибровка порогов; GPU-развёртывание.
